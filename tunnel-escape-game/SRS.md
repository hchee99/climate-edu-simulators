| 항목 | 내용 |
|---|---|
| 프로젝트명 | 지하차도 침수 탈출 게임 (Tunnel Escape) |

## 1. 서론 (Introduction)

### 1.1 목적 (Purpose)
집중호우 시 지하차도(지하공간) 침수의 위험과 올바른 대피 행동요령을, 학생이 직접 플레이하며 체득하도록 하는 교육용 웹 게임의 요구사항을 정의한다.

### 1.2 범위 (Scope)
- 차오르는 물을 피해 출구로 대피하는 단일 스테이지 아케이드 게임
- 핵심 학습: "물이 차면 난간(손잡이)을 잡고 출구로", "골든타임 안에 빠르게 대피"
- 범위 외: 멀티 스테이지, 온라인 랭킹, 서버/DB, 실제 기상 데이터 연동

### 1.3 정의 및 약어
| 용어 | 설명 |
|---|---|
| 손잡이(난간) | 양 끝 길에 있는 안전 구조물. 물에 잠겨도 전진 가능 |
| 골든타임 | 침수 시 안전하게 대피할 수 있는 제한시간 |
| 침수(잠김) | 물 수위가 플레이어 위치까지 차오른 상태 |
| 격자(셀) | 맵을 구성하는 칸 (5열 × 26행) |

### 1.4 참조
- 행정안전부 지하공간 침수 국민행동요령
- 사례: 2022 서울 강남 침수, 2023 오송 지하차도 참사
- 게임 참조: 길건너 친구들(Crossy Road)

## 2. 전체 설명 (Overall Description)

### 2.1 제품 관점
- 별도 백엔드 없는 프론트엔드 단독 웹앱
- HTML Canvas + 순수 JavaScript (외부 게임엔진 없음)
- `index.html` / `style.css` / `game.js` 3파일 구조

### 2.2 제품 기능 요약
1. 5개 길에서 좌우·전진 이동하는 캐릭터 조작
2. 시간에 따라 가속하며 차오르는 물
3. 손잡이·차·나뭇가지·아이템으로 구성된 장애물/보상
4. 골든타임 타이머와 점수
5. 성공/실패 판정 및 안전 행동 메시지

### 2.3 사용자 특성
- 초등학생 (비전문가). 직관적 조작·큰 버튼·그림 위주
- 키보드(방향키) 및 화면 버튼(모바일) 모두 지원

### 2.4 제약 조건
- 브라우저(크롬) 환경, 모바일 터치 고려
- 외부 라이브러리 최소화 (유지보수·이식 용이)

### 2.5 가정 및 의존성
- 물리 계산은 정밀 모델이 아닌 게임용 근사 규칙
- 모든 수치(물 속도·맵 크기 등)는 코드 상단 상수로 분리해 조정 가능

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 맵·이동
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-01 | 맵은 5열 × 26행 격자로 구성하고, 화면은 일부만 보이며 플레이어를 따라 세로 스크롤한다 | 상 |
| FR-02 | 플레이어는 맨 아래에서 시작해 맨 위 출구 도달이 목표다 | 상 |
| FR-03 | 좌/우/앞(위)으로 한 칸씩 이동한다 (방향키 + 화면 버튼) | 상 |
| FR-04 | 과도한 연타를 막기 위해 이동에 쿨다운(약 0.13초)을 둔다 | 중 |

### 3.2 물(침수)
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-05 | 물은 맵 바닥부터 위로 차오른다 | 상 |
| FR-06 | 시작 후 일정 시간(약 1초)은 물이 차지 않는다 (대피 준비 시간) | 중 |
| FR-07 | 물은 시간이 지날수록 점점 빠르게 차오른다 (가속) | 상 |
| FR-08 | 물 수위는 플레이어 발밑→머리 순으로 차오르도록 시각화한다 | 중 |

### 3.3 손잡이 메커니즘 (핵심)
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-09 | 물에 잠긴 상태에서 가운데 길은 위로 전진할 수 없다 | 상 |
| FR-10 | 양 끝 손잡이 길에서는 물에 잠겨도 위로 전진할 수 있다 | 상 |
| FR-11 | 손잡이 길에는 차 장애물이 배치되지 않는다 | 상 |

### 3.4 장애물·아이템
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-12 | 버려진 차는 해당 칸을 막아 지나갈 수 없다 | 상 |
| FR-13 | 차는 가운데 3칸에 줄마다 엇갈리게 배치되어 지그재그 이동을 유도한다 | 중 |
| FR-14 | 나뭇가지는 가운데를 좌우로 떠다니며, 닿으면 한 칸 뒤로 밀린다 | 중 |
| FR-15 | 차와 나뭇가지는 서로 최소 1칸 이상 간격을 둔다 | 하 |
| FR-16 | 사다리(2칸 위 이동), 튜브(잠시 물 보호) 아이템을 각 1개 배치한다 | 중 |
| FR-17 | 사다리 착지 지점(앞 2칸)에는 장애물을 두지 않는다 | 중 |

### 3.5 판정·UI
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-18 | 출구 도달 시 성공, 골든타임 초과 시 실패로 판정한다 | 상 |
| FR-19 | 물에 잠기는 것 자체로는 실패하지 않는다 | 상 |
| FR-20 | 골든타임 타이머, 점수, 남은 거리를 화면에 표시한다 | 중 |
| FR-21 | 성공/실패 시 안전 행동요령 메시지를 보여준다 | 중 |
| FR-22 | 시작/다시 버튼으로 게임을 초기화하고 카메라를 시작 위치로 맞춘다 | 중 |

## 4. 비기능 요구사항 (Non-functional Requirements)

| ID | 분류 | 요구사항 |
|---|---|---|
| NFR-01 | 성능 | 60fps에 가깝게 부드럽게 동작 (requestAnimationFrame) |
| NFR-02 | 사용성 | 초등 저학년이 설명 없이 조작 가능 (방향키/큰 버튼) |
| NFR-03 | 호환성 | 크롬 데스크톱 및 모바일 터치 동작 |
| NFR-04 | 유지보수성 | 물 속도·맵 크기·쿨다운 등 밸런스 값을 코드 상단 상수로 분리 |
| NFR-05 | 배포성 | 정적 호스팅(GitHub Pages)으로 링크 시연 가능 |
| NFR-06 | 확장성 | 에셋(이미지·소리) 추가 및 레벨 확장이 용이한 구조 |

## 5. 외부 인터페이스 요구사항

### 5.1 사용자 인터페이스 (UI)
- 상단 HUD: 골든타임, 점수, 남은 거리, 상태 메시지
- 중앙: 게임 캔버스 (단면 터널, 물, 캐릭터, 장애물)
- 하단: ← ↑ → 이동 버튼 + 시작/다시 버튼 + 범례

### 5.2 입력 인터페이스
| 입력 | 동작 |
|---|---|
| 방향키 ← → ↑ | 좌/우/전진 |
| 화면 버튼 | 동일 (모바일) |

### 5.3 소프트웨어 인터페이스
- 외부 라이브러리 없음 (브라우저 Canvas API만 사용)

## 6. 밸런스 파라미터 (조정 가능 상수)

| 상수 | 의미 | 현재값 |
|---|---|---|
| `WROWS` | 맵 세로 길이(행) | 26 |
| `TOTAL` | 골든타임(초) | 20 |
| `W_DELAY` | 시작 후 물 유예(초) | 1.0 |
| `W_BASE` | 물 시작 상승속도(행/초) | 2.7 |
| `W_ACC` | 물 가속 | 0.13 |
| `MOVE_CD` | 이동 쿨다운(초) | 0.13 |
| `SAFE_ROWS` | 시작 부근 장애물 없는 칸 | 3 |
